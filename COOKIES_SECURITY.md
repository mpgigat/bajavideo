# Seguridad de Cookies - Guía Completa

## ⚠️ IMPORTANTE: NUNCA SUBAS cookies.txt A UN REPOSITORIO PÚBLICO

El archivo `cookies.txt` contiene información sensible de tu cuenta de YouTube.

## Opciones Seguras

### Opción 1: Repositorio Privado (Más Simple)

1. Haz tu repositorio privado en GitHub/GitLab
2. Sube cookies.txt al repositorio
3. Configura Coolify para acceder al repositorio privado

**En Coolify:**
- Ve a "Git Source"
- Agrega tu SSH key o Personal Access Token
- Selecciona el repositorio privado

### Opción 2: Variables de Entorno (Más Seguro - Recomendado)

Esta opción NO requiere subir cookies.txt al repositorio.

#### Paso 1: Crear cookies.txt localmente

Usa la extensión de Chrome como se describe en [`COOKIES_GUIDE.md`](COOKIES_GUIDE.md:1)

Guarda el archivo en tu computadora local (no en el repositorio):
```
C:\Users\Sena\cookies.txt
```

#### Paso 2: Leer el contenido del archivo

Abre el archivo `cookies.txt` con un editor de texto y copia todo el contenido.

#### Paso 3: Configurar en Coolify

1. Ve a tu servicio en Coolify
2. Ve a "Environment Variables"
3. Agrega una nueva variable:
   - **Key:** `YOUTUBE_COOKIES`
   - **Value:** (pega todo el contenido de cookies.txt)

El contenido se verá algo así:
```
# Netscape HTTP Cookie File
# This file is generated by yt-dlp.  Do not edit.

.youtube.com	TRUE	/	FALSE	1735689600	VISITOR_INFO1_LIVE	...
.youtube.com	TRUE	/	FALSE	1735689600	PREF	...
.youtube.com	TRUE	/	FALSE	1735689600	CONSENT	...
...
```

#### Paso 4: Hacer Redeploy

1. Haz clic en "Redeploy" en Coolify
2. Espera a que se complete

#### Paso 5: Verificar

Revisa los logs en Coolify, deberías ver:
```
Created cookies file from environment variable
Using cookies file for authentication
```

## Ventajas de Variables de Entorno

✅ **Más seguro** - Las cookies no están en el repositorio
✅ **Repositorio puede ser público** - No hay información sensible
✅ **Fácil de actualizar** - Solo cambias la variable en Coolify
✅ **Rotación simple** - Puedes cambiar cookies sin modificar código

## Actualizar Cookies

Cuando las cookies expiren (30-60 días):

1. Genera un nuevo cookies.txt localmente
2. Copia el contenido
3. Ve a Coolify → Environment Variables
4. Edita `YOUTUBE_COOKIES` con el nuevo contenido
5. Haz redeploy

## Verificación de Seguridad

### Para confirmar que cookies.txt no está en el repositorio:

```bash
# Verificar que está en .gitignore
git check-ignore -v cookies.txt

# Debería mostrar:
# .gitignore:3:cookies.txt
```

### Para ver qué archivos se subirán:

```bash
git status
```

No deberías ver `cookies.txt` en la lista.

## Código Actualizado

El código en [`app.py`](app.py:46) ahora:
1. Busca cookies en variable de entorno `YOUTUBE_COOKIES`
2. Si existe, crea el archivo cookies.txt automáticamente
3. Si no existe variable, busca el archivo cookies.txt
4. Si no hay cookies, continúa sin autenticación

## Soporte en Coolify

Coolify soporta variables de entorno de forma nativa:
- Ve al servicio
- Environment Variables
- Agrega `YOUTUBE_COOKIES` con el contenido
- Guarda y redeploy

## Preguntas Frecuentes

**¿Las cookies en variables de entorno son seguras?**
Sí, Coolify cifra las variables de entorno y no las muestra en logs.

**¿Puedo usar el repositorio público con variables de entorno?**
Sí, las cookies no están en el código, solo en la configuración de Coolify.

**¿Cómo sé si las cookies expiraron?**
Verás el mismo error de "Sign in to confirm you're not a bot" en los logs.

**¿Puedo tener múltiples cuentas de YouTube?**
Sí, pero necesitarías rotar las cookies manualmente en la variable de entorno.
